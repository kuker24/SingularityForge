# Obsidian Sync

## Purpose

Use Obsidian as durable project memory while keeping token usage low.

## Rules

1. Search notes before reading full notes.
2. Read only relevant files.
3. Write concise session logs.
4. Do not store secrets.
5. Keep architecture decisions in ADR format.

## Vault Structure

```txt
00-Inbox/
01-Projects/
02-Architecture/
03-Standards/
04-Skills/
05-Session-Logs/
06-Debug-Notes/
07-Release-Notes/
99-Archive/
```

## Output

```txt
Notes read:
Notes written:
Summary:
Next memory action:
```
