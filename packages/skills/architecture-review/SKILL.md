# Architecture Review

## Review Areas

1. Product goal and constraints.
2. System boundaries.
3. Data flow.
4. Module responsibilities.
5. State ownership.
6. Error handling.
7. Security boundary.
8. Testing strategy.
9. Deployment and operations.
10. Migration risk.

## Method

- Read only architecture-relevant files first.
- Compare implementation against stated goals.
- Separate confirmed evidence from inference.
- Propose minimal improvements before large rewrites.
